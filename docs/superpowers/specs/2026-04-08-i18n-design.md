# UI Kit — 다국어(i18n) 지원 설계

## 개요

UI Kit 데모 사이트와 컴포넌트를 한국어(ko)/영어(en) 두 가지 언어로 지원한다. URL 기반 locale prefix로 언어를 전환하며, react-i18next를 사용한다.

### 목표

- `/:locale/components/...` URL 구조로 언어별 페이지 제공
- 기본 locale: 한국어 (ko)
- 컴포넌트 내부 기본 텍스트도 locale-aware
- 기존 컴포넌트 API 호환성 유지

---

## 1. 번역 파일 구조

```
src/locales/
├── ko/
│   ├── common.json
│   └── autocomplete.json
└── en/
    ├── common.json
    └── autocomplete.json
```

### namespace 분리 원칙

- `common` — 사이드바 타이틀, 로딩 텍스트, 언어 전환 UI 등 공통 문자열
- 컴포넌트마다 하나의 namespace — 새 컴포넌트 추가 시 해당 namespace JSON만 추가

## 2. i18n 초기화

`src/i18n.ts` 파일에서 i18next를 초기화한다.

- `i18next` + `react-i18next`의 `initReactI18next` 플러그인 사용
- `resources`에 ko/en JSON을 직접 import (2개 언어이므로 번들에 포함, 별도 fetch 불필요)
- `fallbackLng: 'ko'`
- `interpolation.escapeValue: false` (React가 XSS 방어)
- `ns: ['common', 'autocomplete']`, `defaultNs: 'common'`
- `main.tsx`에서 `import './i18n'`으로 초기화

## 3. 라우팅 구조 변경

### URL 패턴

```
/:locale/components/autocomplete   ← 메인 라우트
/                                  ← /ko/components/autocomplete로 리다이렉트
```

### App.tsx 라우트 구조

```tsx
<Routes>
  <Route path="/" element={<Navigate to="/ko/components/autocomplete" replace />} />
  <Route path="/:locale/components" element={<LocaleLayout />}>
    {componentRoutes.map(...)}
  </Route>
</Routes>
```

### LocaleLayout

`DocsLayout`을 감싸는 래퍼 컴포넌트:
- URL의 `:locale` 파라미터를 읽어 `i18n.changeLanguage(locale)` 호출
- 유효하지 않은 locale이면 `/ko/...`로 리다이렉트
- 기존 `DocsLayout`은 레이아웃 역할만 유지

### 언어 전환 UI

`DocsLayout` 사이드바 상단에 언어 전환 링크 추가:
- `KO | EN` 형태
- 현재 locale 하이라이트 (볼드)
- 클릭 시 현재 경로의 locale 부분만 교체하여 이동

## 4. 컴포넌트 i18n 통합

### Autocomplete 컴포넌트

`useTranslation('autocomplete')`를 사용하여 내부 기본 텍스트를 번역한다.

**변경되는 기본값:**
- `noResultsMessage` — props 미전달 시 `t('component.noResults')`
- `Loading...` 텍스트 — `t('component.loading')`
- `aria-live` 텍스트 — `t('component.resultCount', { count })`

**기존 API 호환성:**
- props로 명시적 값을 전달하면 번역 키보다 우선 적용
- `noResultsMessage="Custom"` 형태로 쓰면 i18n 무관하게 "Custom" 표시

### AutocompletePage

- 모든 하드코딩된 텍스트를 `useTranslation('autocomplete')`의 `t()` 호출로 교체
- placeholder도 번역 키에서 가져옴

## 5. 테스트 전략

### 유닛 테스트

- 테스트 setup에서 i18n 초기화 (메모리 리소스, 기본 locale 고정)
- Autocomplete 테스트: locale `'ko'`와 `'en'`에서 기본 텍스트가 올바르게 렌더링되는지 확인
- 기존 테스트는 i18n 초기화 추가 외에 변경 최소화

### E2E 테스트

- `/ko/components/autocomplete` 페이지 로드 및 인터랙션 테스트
- `/en/components/autocomplete` 페이지 로드 및 인터랙션 테스트
- 언어 전환 링크 클릭 → URL 변경 + 텍스트 변경 검증
- axe 접근성 스캔은 양쪽 locale에서 실행
